# Gemini Context Review & Understanding

## 1. Project Overview & Goal
We are building a modern, data-driven website for **Cogesto Consulting** using the **Astro** framework. The primary objective is to create high-quality, compliant pages by repurposing structure and content strategies from top-tier consulting firms (like Wood Mackenzie or Bain), adapted specifically for Cogesto's brand and context.

## 2. The "Content-First" Workflow
I understand the process to be as follows:

1.  **Scrape & Extract:**
    *   Use the scraper scripts (e.g., in `content_factory/` or `webpage_to_markdown.py`) to extract content from target inspirational URLs.
    *   *Goal:* Get raw material to shape the narrative.
2.  **Contextualize:**
    *   Merge extracted content and adapt it for Cogesto Consulting.
    *   **Crucial Resource:** `cogesto_context.txt` (generated by `get_cogesto_context.py`) provides the ground truth for Cogesto's mission, values, and tone.
3.  **Structure Selection:**
    *   **Source of Truth:** Use files in `page_structure/**` (e.g., `bainservice_pagestructure.md`, `service1_pagestructure.md`) as the architectural blueprints.
    *   *Directive:* Do **not** use `Ref_html/**` for structure decisions; rely on the analyzed `page_structure` files.
4.  **Component Mapping:**
    *   Map the chosen structure to components listed in `COMPONENTS_INVENTORY.md`.
    *   **Rule:** Prioritize "Fully Compliant" components. If a component is missing or non-compliant, create or refactor it following `COMPONENT_DEVELOPMENT_PROTOCOL.md`.
5.  **Implementation:**
    *   Centralize text content in `src/content/fr.ts` (or similar data files).
    *   Build the `.astro` page, importing content props and passing them to components.
    *   **Style:** Adhere strictly to `DESIGN_SYSTEM.md` and `src/styles/styles.css` (Global CSS, BEM naming, no scoped styles).

## 3. Codebase Architecture Understanding
*   **Framework:** Astro + TypeScript.
*   **Styling:** Global CSS (`src/styles/styles.css`). Components are style-agnostic (pure HTML structure + classes).
*   **Data Strategy:** Content is decoupled from presentation. Components receive data via `Astro.props`.
*   **Key Directories:**
    *   `src/components/`: Reusable UI blocks.
    *   `src/content/`: Data sources (`fr.ts`).
    *   `page_structure/`: Layout blueprints.
    *   `content_factory/`: Python scripts for migration/scraping.
